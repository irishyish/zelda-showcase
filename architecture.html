The Seven Layers

Each layer in ZELDA’s architecture has a distinct purpose, clear responsibilities and explicit boundaries. By compartmentalising cognition in this way, the system avoids hidden feedback loops, respects user consent and ensures that ethical constraints are applied before any output is delivered. Explore each layer below:

Layer 1: Input & Context Boundary

Purpose: Control what the system is allowed to know.

Responsibilities: Accept user input and explicitly scoped context, enforce consent boundaries and prevent hidden state accumulation.

Key constraint: No long‑term user profiling or silent data harvesting — reflecting privacy concerns about AI systems that reuse personal data without consent【709742148798799†L82-L97】.

Layer 2: Cognitive Core

Purpose: Generate candidate understandings and actions.

Responsibilities: Interpret inputs, generate hypotheses and produce candidate action plans.

Design characteristics: Time‑aware reasoning across multiple horizons and non‑executable candidate generation to keep planning separate from execution.

Layer 3: Reflection & Audit

Purpose: Detect errors, contradictions and risks.

Responsibilities: Evaluate candidate actions for coherence, identify contradictions, flag ethical or safety concerns and ensure evidence thresholds are met.

Hard limits: A fixed iteration count to avoid infinite self‑reflection and tool spam; explicit exit conditions stop the loop when conditions are satisfied【669129473536362†L292-L359】.

Layer 4: Ethical Constraint Engine

Purpose: Enforce moral and design invariants.

Structure: A hybrid of non‑negotiable constraints and priority‑weighted ethical principles that veto manipulative, harmful or coercive behaviours.

Example constraint: The system cannot manipulate user behaviour — a principle supported by scholarship on the importance of respecting user autonomy and avoiding hidden persuasion【813286105083797†L170-L183】【538976851382273†L392-L437】.

Layer 5: Decision Authority & Commit

Purpose: Terminate deliberation and commit to one outcome.

Responsibilities: Select a satisficing (good‑enough) action rather than a theoretically optimal one; enforce finality; prevent post‑hoc rationalisation.

Mechanism: Satisficing thresholds ensure that decisions occur within defined time and quality bounds.

Layer 6: Execution Interface

Purpose: Translate decisions into outputs or actions.

Characteristics: Minimal transformation, deterministic mapping from decisions to outputs, and an “output‑only” posture that prevents hidden side effects.

Safety posture: Execution never changes the internal ethical substrate — reflecting concerns about self‑modifying AI which can create unpredictable behaviours without oversight【904532470556632†L475-L593】.

Layer 7: Oversight & Update Plane

Purpose: Preserve long‑term coherence and authority.

Authorities: Only the creators or authorised overseers may modify core cognition, update ethical constraints or audit historical decisions.

Capabilities: Manage version control, apply kill‑switches, prevent drift and ensure that any changes are transparent and reversible【904532470556632†L475-L593】【361354759294151†L571-L585】.
